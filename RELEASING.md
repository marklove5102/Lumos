# Releasing a Lumos Game

## Prerequisites

- macOS with Xcode installed
- `Tools/premake5` binary present
- Runtime built in Release mode:
  ```bash
  ./Scripts/MacOS/BuildMacOS.sh
  ```

---

## 1. Build Asset Pack

Packs all project assets into a single `Assets.lpak` file for distribution.

**Option A — Editor UI:**
- Open your project in the Editor
- File menu → **Build Asset Pack**
- Wait for the "Asset pack built: Assets.lpak" notification
- `Assets.lpak` appears in your project root

**Option B — CLI:**
```bash
./bin/Release-macosx-x86_64/Runtime.app/Contents/MacOS/Runtime \
    --project=ExampleProject \
    --pack-assets=ExampleProject/Assets.lpak
```
Exit 0 = success, exit 1 = failure.

Default excludes: `Cache/`, `Imported/` subdirectories and `.meta` files.

---

## 2. macOS Packaging

```bash
./Scripts/MacOS/PackageGame.sh ExampleProject
```

This script:
1. Prepares icons from `Assets/Textures/icon.png` if present
2. Packs assets into `ExampleProject/Assets.lpak` (requires Release Runtime binary)
3. Generates a macOS Xcode project via premake5
4. Opens Xcode — archive and export from there

The `.lpak` is bundled inside the `.app` automatically (it's in the project folder which premake copies as a resource). The Runtime auto-mounts it on launch.

To remove `Assets/` from the bundle and ship only the `.lpak`, add an Xcode build phase to delete it post-copy, or adjust the premake resource copy rules.

---

## 3. iOS Packaging

```bash
./Scripts/MacOS/PackageGame.sh ExampleProject --ios
```

Same flow as macOS but generates an iOS Xcode project. Open the generated project, set your team/bundle ID, and archive.

Assets.lpak is included in the bundle Resources if it exists in the project dir at premake time.

---

## 4. Renaming the Executable

In `premake5.lua`, find the `Runtime` project block and change `targetname`:

```lua
project "Runtime"
    targetname "YourGameName"   -- change this
```

Re-run premake after changing.

---

## 5. App Store Submission Checklist

- [ ] Bundle ID set in `.lmproj` (`BundleIdentifier` field) or Xcode project settings
- [ ] Version and build number set in Xcode
- [ ] App category set (Xcode → Target → Info → Application Category)
- [ ] `ITSAppUsesNonExemptEncryption = NO` in Info.plist (handled by premake template)
- [ ] Privacy manifest (`PrivacyInfo.xcprivacy`) present if using tracked APIs
- [ ] Icons provided for all required sizes (use `Scripts/MacOS/PrepareIcon.sh`)
- [ ] Launch screen configured (auto-generated by `Scripts/MacOS/GenerateLaunchScreen.sh`)
- [ ] Tested on device (not just simulator)
- [ ] Archive → Validate App → Distribute App (App Store Connect)
